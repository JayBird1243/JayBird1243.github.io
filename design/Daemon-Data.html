Daemon Data

← Back to Design

Daemon Data was started as a meme stock market trend analysis tool for personal use. While coding a locally hosted version, I came across a string of reddit posts offering payment to any app which performed the functions my app was already performing.

I moved to transform the app into a scaleable hosted SaaS tool. I used Firebase for most of the tech stack, which allowed for rapid development and scaling. Unfortunately, Reddit has recently cracked down on API usage, and my project has stalled while I await potential official data licensure.

The app was programmed using the Cursor IDE.

System Architecture

Daemon Data consisted of three main components: Data collection, preprocessing/analysis, and visualization/notification.

Data Collection

Data is collected from Reddit using their official API, and stored in a Google Firestore database. In order to optimize data storage in a user-scaleable manner, post data is stored centrally, and users are given 'claims' on posts they've requested or scraped.

Post data is either gathered in a single operation, or on a loop, allowing for the user to track engagement over time. The loop is created with a Google Cloud Scheduler, sending a Pub-Sub message for each selected subreddit.

Preprocessing & Analysis

Once raw data is ingested, the preprocessing engine normalizes the text and metadata. This involves:

  • Sentiment analysis on post titles and bodies.
  • Keyword extraction to identify trending tickers or topics.
  • Statistical aggregation to determine momentum and velocity of discussion.

This data is then structured to allow for efficient querying and trend identification.

Visualization & Notification

The frontend provides users with actionable insights through:

  • Real-time dashboards displaying trending topics.
  • Customizable alerts triggered by specific keywords or sentiment thresholds.
  • Historical data charts to visualize trend evolution over time.
